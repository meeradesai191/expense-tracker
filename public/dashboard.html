How this proves the pipeline works

During the GitHub Actions run, the workflow reads the current commit SHA and timestamp and passes them into docker build --build-arg. The Dockerfile bakes those values in as environment variables, so the values above always describe exactly which commit is currently live — no manual updating, no guessing.

DevOps Pipeline: Git → GitHub Actions (CI) → Docker (Build) → Deploy
