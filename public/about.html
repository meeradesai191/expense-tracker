About this project

A small Expense Tracker app used as a vehicle to demonstrate DevOps automation concepts for an MCA Semester 4 seminar. The app itself is intentionally simple — the focus is the pipeline around it.

Pipeline stages (.github/workflows/ci.yml)

1
Lint
ESLint checks code style
2
Test
Jest + Supertest run automated tests
3
Build
Docker image built with commit metadata
4
Push
Image pushed to Docker Hub registry
5
Deploy
Render auto-pulls and deploys the new image

Why this counts as DevOps automation

No step above is run by hand. A single git push to the main branch triggers everything: tests must pass before an image is even built, the image is tagged with the exact commit it came from, and the live site updates itself — which is exactly what the Dashboard page is there to prove.

Tech stack

Runtime: Node.js 18, Express
Testing: Jest, Supertest
Linting: ESLint
Containerization: Docker (multi-stage build)
CI/CD: GitHub Actions
Hosting: Render (free tier, deploys straight from a Docker image)

DevOps Pipeline: Git → GitHub Actions (CI) → Docker (Build) → Deploy
